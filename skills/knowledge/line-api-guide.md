## ステータス

**検討中** - 将来の拡張オプション

## 概要

LINEをフライデーの入出力インターフェースとして使用する可能性。

## LINE API の種類

| API | 用途 | 料金 |
|-----|------|------|
| **LINE Messaging API** | ボットでメッセージ送受信 | 無料枠あり（月200通まで無料） |
| **LINE Notify** | 通知を送るだけ | **完全無料** |
| **LINE Login** | LINEアカウントでログイン | 無料 |

## 活用案

### パターン1: 通知専用（LINE Notify）

```
フライデー処理完了
    ↓
LINE Notify で通知
    ↓
ユーザーのLINEに結果が届く
```

**メリット:** 簡単・無料
**デメリット:** 一方通行（フライデーへの指示はできない）

### パターン2: 双方向（Messaging API）

```
ユーザー（LINE）: 「明日の予定教えて」
    ↓
LINE Webhook → フライデーAPI
    ↓
フライデー: 処理実行
    ↓
Messaging API → LINEに返信
```

**メリット:** LINEから直接指示できる
**デメリット:** 月200通超えると課金、Webhook設定が必要

### パターン3: ブラウザUI + LINE通知

```
ユーザー: ブラウザUIで指示
    ↓
フライデー: 処理実行
    ↓
結果をLINE Notifyで通知
```

**メリット:** 両方の良いとこ取り
**デメリット:** 2つのインターフェース管理

## 実装に必要なもの

### LINE Notify（簡単）

1. LINE Notify トークン取得
2. FastAPIに通知エンドポイント追加
3. 処理完了時にLINEへPOST

### Messaging API（本格的）

1. LINE Developers でチャネル作成
2. Webhook URL設定（Cloudflare Tunnel経由）
3. メッセージ受信→処理→返信のフロー実装
4. リッチメニュー設定（オプション）
